# 快速启用新浪财经API

## 步骤1：创建环境变量文件

在项目根目录创建 `.env.local` 文件：

```bash
# 在项目根目录执行
touch .env.local
```

## 步骤2：配置环境变量

编辑 `.env.local` 文件，添加以下内容：

```env
# 启用新浪财经真实API
USE_REAL_API=true
```

## 步骤3：重启开发服务器

```bash
# 停止当前服务器（Ctrl+C）
# 然后重新启动
npm run dev
```

## 验证

启动后，访问 http://localhost:3000，你应该能看到：

1. ✅ 真实的股票涨跌幅排行榜数据
2. ✅ 实时更新的股票价格
3. ✅ 真实的股票名称和代码

如果看到的是模拟数据，请检查：

1. `.env.local` 文件是否存在
2. `USE_REAL_API=true` 是否正确设置
3. 开发服务器是否已重启
4. 查看控制台是否有错误信息

## 故障排查

### 问题：仍然显示模拟数据

**解决方案**：
1. 确认 `.env.local` 文件在项目根目录（与 `package.json` 同级）
2. 确认文件内容为：`USE_REAL_API=true`（没有引号）
3. 完全重启开发服务器（停止后重新运行 `npm run dev`）

### 问题：API返回空数据

**可能原因**：
- 网络连接问题
- 新浪财经API接口临时不可用
- 请求频率过高

**解决方案**：
1. 检查网络连接
2. 查看浏览器控制台和服务器终端的错误信息
3. 等待几分钟后重试
4. 如果持续失败，可以暂时切换回模拟数据（设置 `USE_REAL_API=false`）

### 问题：CORS错误

**说明**：所有API调用都在Next.js服务器端进行，不会出现CORS问题。如果看到CORS错误，可能是：
- 在客户端组件中直接调用了API
- 检查代码是否正确使用了API路由（`/api/stocks/ranking`）

## 切换回模拟数据

如果需要切换回模拟数据：

```env
# .env.local
USE_REAL_API=false
```

或者直接删除 `USE_REAL_API` 这一行。
